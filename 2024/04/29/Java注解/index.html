注解是什么

注解(Annotation)也叫元数据，用于对代码进行说明，可以对包、类、接口、字段、方法参数、局部变量等进行注解。本质上就是代码里的特殊标志，这些标志可以在编译，类加载，运行时被读取，并根据这些信息执行相应的处理，以便于其他工具补充信息或者进行部署。

如果说注释是写给人看的，那么注解就是写给程序看的。它更像一个标签，贴在一个类、一个方法或者字段上。它的目的是为当前读取该注解的程序提供判断依据。比如程序只要读到加了@Test的方法，就知道该方法是待测试方法，又比如@Before注解，程序看到这个注解，就知道该方法要放在@Test方法之前执行。

注解的类型

1.元注解

元注解用于定义注解，包括四种：@Retention、@Target、@Inherited、@Documented

@Retention：用于表明注解被保留的阶段，其取值包括：

  • RetentionPolicy.SOURCE - 注解信息保留在xxx.java源文件中，编译后丢弃，即不保留在xxx.class文件中
  • RetentionPolicy.CLASS - 注解信息保留在xxx.class文件中，但不会加在到jvm中，该值为系统默认值
  • RetentionPolicy.RUNTIME - 注解信息会保留在源文件、类文件中，在执行的时也加载到Java的JVM中，因此可以通过反射进行读取

@Target：用于表明注解使用的范围，包括类，接口，方法，变量等

@Inherited：添加该注解表明注解可以被子类继承

@Documented：添加该注解表明生成javadoc文档

2.java自带注解

Java自带的标准注解，包括@Override（标明重写某个方法）、@Deprecated（标明某个类或方法过时）和@SuppressWarnings（标明要忽略的警告），使用这些注解后编译器就会进行检查。

3.自定义注解

即根据用户需求实现注解。

注解的实现

实现注解的步骤

  1. 声明注解，设置属性并设置default值

  2. 实现注解处理器。借助反射，获取注解对象，读取注解属性值, 然后根据注解及属性值做相应处理

  3. 使用注解，可以指定属性的值，不指定时取默认值。

  4. 获取添加了注解的目标。通常是Class对象，Method对象，Field对象，还有Constructor对象，Parameter对象，Annotation对象等

    a.通过已知对象，获取Class对象

    b.通过全类路径，获取Class对象
